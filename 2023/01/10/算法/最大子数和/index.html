最大子数和

题目

给你一个整数数组nums，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。子数组是数组中的一个连续部分。

方法一

分析

这是一道动态规划的问题，那我们就按照动态规划的步骤来解决这道题。首先我们通过暴力枚举，来找找看这道题的规律。

我们以nums = [-2,1,-3,4,-1,2,1,-5,4]为例。最直观的求最大子数和的方法是，从nums[0]开始，计算它的子数和，那分别有9种情况：第一种sums1=nums[0]，第二种sums2=nums[0]+nums[1]，第三种sums2=nums[0]+nums[1]+nums[2]…nums[2]有8种情况sums1=nums[2]，第二种sums2=nums[2]+nums[3]…那其实我们可以大致感觉出来，大致的规律是$f(n)=f(n-1)+nums[i]$。

当如果这部分的子数是一直正数的话，这规律是成立的。那还有一种情况，一部分子数是一直为负数，那这样时候，前面的数都是累赘，最大的子数和其实就是nums[i]自身。那经过修正，状态转移方程其实就是$f(i)=max\left { f(i−1)+nums[i],nums[i] \right } $，那边界条件当然就是$f(0)=nums[0]$这个特殊情况了。

  • 时间复杂度：$O(n)$，用动态规划的方法，我们只用遍历一遍数组即可求出最大子数和
  • 空间复杂度：$O(n)$，我们可以用一个容器存储下来每个$f(i)$的值，然后在计算$f(i)$时直接取出$f(i-1)$的值
    • 这里我的思维惯性让我直接先用哈希表作为这个容器，这样做的好处就是对应关系非常清晰，但是缺点就是很臃肿，时间性能不是很好
    • 然后我想到了用数组存储，因为刚好数组的下标和$f(i)$的值一一对应，可以做到非常轻量
    • 但其实我们可以用前天做的那道题的思路
      • 这道题只考虑到了$f(i)$和$f(i-1)$二者之间的关系，那我用一个变量直接存储pre的值也能满足需求
      • 那空间复杂度就变成了$O(1)$

知识点回顾

map的遍历方式

for-each的方式

  • 在map中其实每一个数据都是用键值对来存储的，那这一对其实对应的数据结构是entry
  • 我们首先将map中的键值对全部转换为entry集合，然后对表的遍历就转化为对entry集合的遍历
  • 基于这种方式可以更近一步
  • 如果我们只用遍历表中的key或者value，我们其实不用把整个键值对全部转换为entry集合

用迭代器进行遍历

  • 值的注意的是，图Map是没有迭代器的（没有继承Iterator接口），只有集合Set才有
  • 因此，我们首先将Map中的键值对全部转换为entry集合，然后用迭代器对其进行遍历
  • 注意：迭代器的类型是Map.Entry

返回数组中的最值

  • 借助 Collections.max()和min() 方法来快速返回数组中的最值。这里以最大值为例

Returns the maximum element of the given collection, according to the natural ordering of its elements. All elements in the collection must implement the Comparable interface. Furthermore, all elements in the collection must be mutually comparable (that is, e1.compareTo(e2) must not throw a ClassCastException for any elements e1 and e2 in the collection).
This method iterates over the entire collection, hence it requires time proportional to the size of the collection.
Params:
coll – the collection whose maximum element is to be determined.
Returns:
the maximum element of the given collection, according to the natural ordering of its elements.

  • 参数：输入的一定是Collection的子类，并且这个子类实现了Comparable接口
    • Collection的子类包括List Set Queue
    • 因此如果想要返回数组中的最值，需要先将数组转化为以上的类型
