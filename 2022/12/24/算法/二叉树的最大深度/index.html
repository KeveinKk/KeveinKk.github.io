二叉树的最大深度

题目

给定一个二叉树，找出其最大深度。二叉树的深度为根节点到最远叶子节点的最长路径上的节点数。

方法一：深度优先搜索

分析

递归的方法是最容易想到的，我们很容易得到关于二叉树最大深度的递归公式：$maxDepth=Max(left,1ight)+1$，其中$left$和$right$指的是节点左右子树的深度。这里的$+1$代表的是当前节点的高度。递归

  • 时间复杂度：$O(N)$，每个节点都会被访问一遍
  • 空间复杂度：$O(height)$，其中$height$指的是树的高度

方法二：广度优先搜索

分析

我们也可以用广度优先搜索，也就是一层一层的遍历的方法来实现这道题：当我们遍历完一层时，那么树的高度就增加一个。

我在之前就知道可以用队列来实现树的广度优先遍历，但这次是自己第一次来动手实现这个功能，我不太清楚如何来遍历一层：是每遍历一层就新申请一个队列吗？每遍历到一个节点，就把它的左右孩子放到下一层新的队列里，或者是别的方法。

我们肯定不需要每遍历一层就申请一个新的队列。我们可以设置一个变量sizeOfQue来记录每一层节点的个数：所有的一层的节点都添入队列时，我们就能统计到这一层有多少个节点，每出一个节点sizeOfQue数就减少一个，同时这个节点的左右孩子都入队。当sizeOfQue为0时，就代表上一层所有的节点都已经出队，队伍里剩下的都是下一层的节点，那我们统计一下队伍里有多少个节点，就又可以继续下去…

  • 时间复杂度：$O(N)$，我们需要把所有节点都访问一遍
  • 空间复杂度：$O(N)$，我们需要设置一个队列来存储一层的节点，最坏情况下所有的节点都会在一层（除了根节点）

Queue

Queue接口

然后由于要用到队列的特性，我们就考虑使用Java中队列的实现类。

Queue接口在java.util包下，它继承了Collection接口，它的特性自然不用说：只能在队尾入队，对头出队，并且是先进先出。

The Queue interface is present in java.utilcessed in FIFO(First In First Out) order. It is an ordered list of objects with its use limited to inserting elements at the end of the list and deleting elements from the start of the list, (i.e.), it follows the FIFO or the First-In-First-Out principle.

Queue中常用的方法

boolean add(E e)

Inserts the specified element into this queue if it is possible to do so immediately without violating capacity restrictions, returning true upon success and throwing an IllegalStateException if no space is currently available.

  • 添加一个元素到队列中
  • 如果添加成功返回true，当空间不足添加失败时抛出异常，但不会返回false
boolean offer(E e)

Inserts the specified element into this queue if it is possible to do so immediately without violating capacity restrictions. When using a capacity-restricted queue, this method is generally preferable to add(E), which can fail to insert an element only by throwing an exception.

  • 添加元素时一般更推荐使用该方法
    • add()方法在添加失败时只会报错，不会返回异常
    • offer()方法添加成功返回true，添加失败返回false
E remove()

Retrieves and removes the head of this queue. This method differs from pollonly in that it throws an exception if this queue is empty.

  • 移除对头的元素
  • 当队伍为空时，会返回异常
E poll()

Retrieves and removes the head of this queue, or returns null if this queue is empty.

  • 更推荐使用这个方法来移除对头的元素
  • 移除成功返回该元素，移除失败返回null
E peek()

Retrieves, but does not remove, the head of this queue, or returns null if this queue is empty.

  • 返回对头的元素，但对对头的元素不做任何操作
  • 当队伍为空时，返回null

Queue的实现类

作为一个接口，肯定是有具体的类来实现这个接口，在Java中分别有PriorityBlockingQueue、 PriorityQueue和LinkList两个类来实现队列的接口，其中后两个是非线程安全的。

  • PriorityQueue是优先级队列，它不是FIFO的，而是根据元素的优先级，优先级高的或者低的先出队列
  • 我们一般都使用LinkList来具体实现队列
