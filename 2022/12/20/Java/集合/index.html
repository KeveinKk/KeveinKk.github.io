Java学习-集合

集合

概念

  • 集合就是一个容器，可以用来容纳其它类型的数据
  • 作用
    • 集合作为一个载体，可以一次性容纳多个对象
  • 集合中存储的是对象的引用
    • 集合中不能存储基本数据类型
    • 集合中也不能直接存储对象
    • 集合本身也是一个对象，这也意味着，集合中可以存储集合的引用
  • 每一个不同的集合在底层对应不同的数据结构
    • 向不同的集合中增加元素等价于将数据放到了不同的数据结构中
    • 使用不同的集合等价于使用不同的数据结构
    • 根据不同的情况，选择不同的集合
  • 所有的集合类以及集合接口都在java.util下

Collction的继承结构图

  • Java中集合分为两大类
    1. 直接存储数据元素的方式
      • 其超级父类接口是java.util.Collection
    2. 以键/值对的方式存储数据
      • 其超级父类接口是java.util.Map

public interface Iterable<T>

Implementing this interface allows an object to be the target of the enhanced for statement (sometimes called the “for-each loop” statement).

  • 表示可迭代、可遍历的接口
  • 实现这个接口代表集合中所有元素都是可以遍历的
  • 接口中有如下方法：

public interface Iterator<E>

An iterator over a collection. Iterator takes the place of Enumeration in the Java Collections Framework.

  • 集合的迭代器对象
  • 使用迭代器来遍历整个集合
  • 接口中有以下三个方法来实现集合的遍历

List接口

  • 存储元素的特点
    • 有序：存储时的顺序和取出时的顺序一致
    • 可重复：允许在List中存在重复元素
    • 存储的元素有下标：下标从0开始
  • 分别有ArrayList、LinkedList、Vector三个类来实现这个接口

ArrayList

  • 底层采用数组的数据结构
  • 非线程安全的

LinkedList

  • 底层采用双向链表的数据结构

Vector

  • 底层采用数组的数据结构
  • 线程安全的
    • 方法中关键字synchronize修饰
    • 效率较低，使用较少

Set接口

  • 存储元素的特点
    • 无序
    • 不可重复
    • 无下标

继承结构图

Map的继承结构图

  • Map集合与Collection集合无关
  • Map集合中以键值对的方式（key和value）存储元素
    • key和value存储的都是对象的引用
    • key是无序且不可重复的
  • 分别有HashMap、Hashtable两个类来实现这个接口
  • 有一个子接口SortedMap，TreeMap实现这个子接口
    • SortedMap接口中的key同样是无序且不可重复的
    • 且集合key部分的元素会自动按照大小进行排序
    • 区分这里的排序和List集合中的有序的区别
      • 排序是指：从大到小进行排序
      • 有序是指：存的顺序和取的顺序一致

HashMap

  • 底层采用哈希表的数据结构
  • 非线程安全的

Hashtable

  • 底层采用哈希表的数据结构
  • 线程安全的
    • 效率较低，使用较少
  • 有一个子类Proprieties
    • key和value只支持String类型

TreeMap

  • 底层采用二叉树的数据结构
