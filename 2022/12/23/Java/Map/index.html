Map接口

发表于 2022-12-23 分类于 Java
Java学习-Map

Map

Map接口

  • Map集合和Collection集合没有关系
    • Map集合以键值对（key和vlaue）存储元素
      • 其中key和value存储的都是对象的引用
      • key起主导作用，value只是其附属品
    • Map集合的特点
      • 无序不可重复
  • 有三个类直接实现了Map接口
    • HashMap：底层是哈希表数据结构
    • Hashtable：底层同样是哈希表数据结构，但是因为其是线程安全的，效率较低
      • Properties：继承Hashtable，但是key和value存储的对象都是字符串
    • SortedMap：集合key部分元素会自动排序
      • TreeMap：底层数据结构是一个二叉树

Map接口中的常用方法

V put(K key, V value)

Associates the specified value with the specified key in this map (optional operation). If the map previously contained a mapping for the key, the old value is replaced by the specified value. (A map m is said to contain a mapping for a key k if and only if m.containsKey(k) would return true.

Returns:

the previous value associated with key, or null if there was no mapping for key. (A null return can also indicate that the map previously associated null with key, if the implementation supports null values.)

  • 向Map集合添加键值对
  • 返回值：
    • 返回的是这个key之前绑定的value
    • 如果之前没有绑定过value则返回null

V get(Object key)

Returns the value to which the specified key is mapped, or null if this map contains no mapping for the key.

  • 通过key去获得value

void clear()

Removes all of the mappings from this map (optional operation). The map will be empty after this call returns.

  • 清空Map集合

V remove(Object key)

Removes the mapping for a key from this map if it is present (optional operation). More formally, if this map contains a mapping from key k to value v such that (key==null ? k==null : key.equals(k)), that mapping is removed. (The map can contain at most one such mapping.)

Returns:

the previous value associated with key, or null if there was no mapping for key.

  • 通过key来删除Map中的键值对
  • 返回删除的key所对应的value，如果不存在key时返回null

boolean containsKey(Object key)

Returns true if this map contains a mapping for the specified key. More formally, returns true if and only if this map contains a mapping for a key k such that (key==null ? k==null : key.equals(k)). (There can be at most one such mapping.)

  • 判断Map中是否包含某个key

boolean containsValue(Object value)

Returns true if this map maps one or more keys to the specified value. More formally, returns true if and only if this map contains at least one mapping to a value v such that (value==null ? v==null : value.equals(v)). This operation will probably require time linear in the map size for most implementations of the Map interface.

  • 判断Map中是否包含某个value
  • 底层是用equals方法
    • 需要重写equals方法

boolean isEmpty()

Returns true if this map contains no key-value mappings.

  • 判断某个Map是否为空

int size()

Returns the number of key-value mappings in this map. If the map contains more than Integer.MAX_VALUE elements, returns Integer.MAX_VALUE.

  • 返回Map中键值对的个数

Collection<V> values()

Returns a Collection view of the values contained in this map. The collection is backed by the map, so changes to the map are reflected in the collection, and vice-versa. If the map is modified while an iteration over the collection is in progress (except through the iterator's own remove operation), the results of the iteration are undefined. The collection supports element removal, which removes the corresponding mapping from the map, via the Iterator.remove, Collection.remove, removeAll, retainAll and clear operations. It does not support the add or addAll operations.

  • 获取Map集合中所有的vlaue，返回一个Collction

Set<K> keySet()

Returns a Set view of the keys contained in this map. The set is backed by the map, so changes to the map are reflected in the set, and vice-versa. If the map is modified while an iteration over the set is in progress (except through the iterator's own remove operation), the results of the iteration are undefined. The set supports element removal, which removes the corresponding mapping from the map, via the Iterator.remove, Set.remove, removeAll, retainAll, and clear operations. It does not support the add or addAll operations.

  • 获取Map集合中所有的key

Set<Map.Entry<K,V>> entrySet()

Returns a Set view of the mappings contained in this map. The set is backed by the map, so changes to the map are reflected in the set, and vice-versa. If the map is modified while an iteration over the set is in progress (except through the iterator's own remove operation, or through the setValue operation on a map entry returned by the iterator) the results of the iteration are undefined. The set supports element removal, which removes the corresponding mapping from the map, via the Iterator.remove, Set.remove, removeAll, retainAll and clear operations. It does not support the add or addAll operations.

  • 将整个Map变成一个Set集合
    • 这个集合中的数据类型是Map.Entry<K,V>

遍历Map的方法

方法一

  • 先通过Set<K> keySet()获得Map所有的key，然后再通过get()方法，用key获得对应的value
  • 通过遍历Set集合中的key，来遍历整个Map

方法二

  • 直接用Set<Map.Entry<K,V>> entrySet()方法，将整个Map变成集合
  • 然后再遍历这个集合
    • 集合中的每一项称为节点node
    • 通过getKey()方法获得节点的key
    • 通过getValue()方法获得节点的value
