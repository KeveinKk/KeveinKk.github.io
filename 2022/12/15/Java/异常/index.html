Java学习-异常

异常

异常的概念

  • 程序执行过程中发生了不正常的情况，这种情况被称为异常

  • Java可以对异常进行处理

    • 将异常信息输出到控制台上，供程序员参考
    • 提高程序的健壮性

  • JVM可以在控制台上输出异常信息

异常以类的形式存在

  • 异常在Java中以类的形式存在
  • 每一个异常类都可以创建异常对象

  • 每发生一次异常，都会创建一个该异常类的对象
    • 面向对象编程的思想：
    • 发生即对象！存在即类！

UML

  • Unified Modeling Language同一建模语言
  • 一种图标式语言，是一种开放的方法，用于说明、可视化、构建和编写一个面向对象的、软件密集系统的制品的开放方法
  • 软件设计人员使用UML

异常的继承结构

  • 所有的异常类是从java.lang.Exception类继承的子类。
  • Exception类是Throwable类的子类。除了Exception类外，Throwable还有一个子类Error
    • 意味着无论是异常还是错误都是可以抛出的
    • 所有的错误只要发生，Java都会终止程序的执行，退出JVM
    • 错误是不能处理的
  • Exception类又分为两大类
    • 编译时异常（受控异常）
      • Exception的直接子类
      • 编译时的异常不是在编译阶段发生的，而是说必须在编写程序时预先对这类异常进行处理，如果不处理则无法进行编译！
      • 也就是说运行程序前不可能存在编译异常
      • 发生的概率更高
        • 对于一些发生概率比较高的异常，我们需要在运行之前对其进行处理
    • 运行时异常（非受控异常）
      • RuntimeException
      • 运行时的异常在编写程序时可以处理也可以不处理
      • 发生的概率更低
        • 对于一些发生概率比较低的异常，我们不需要对其进行处理
    • 如果不区分两种异常，即所有异常都需要在编写程序阶段进行预处理
      • 程序是绝对安全的
      • 程序过于复杂，需要处理各种异常
    • 所有的异常都是发生在运行阶段，编译阶段是不会发生异常的。因为编译异常必须在编译时进行处理，否则无法运行。
      • 异常的出现必须实例化异常对象（new），只有在运行时才可以实例化对象

异常的处理

在方法声明的 位置上，使用throws关键字

  • 异常上抛，交给调用者

    • 调用者需要对这个异常进行处理，它处理的方法也是这两种，要么上抛给调用它的上一级，要么自己处理
    • 当上抛到main方法时，main方法继续上抛，上抛给JVM，则JVM只能终止程序的运行

  • 方法声明的异常必须属于编译时异常

  • 因为是编译时异常，所以在调用该方法时必须对这种异常进行预先处理

    • 否则编译报错

用throws关键字对异常处理

  • throws关键字后可以跟多个异常，多个异常之间通过逗号隔开
  • 接受throws抛上来的异常的方法需要对异常进行处理，处理的方法也有两种，要么自己捕获，要么继续向上抛
    • 如果继续向上抛，则抛出的异常必须是接受的异常的父类，或者就是该异常本身
  • main方法一般不再向上抛出异常
    • 异常机制的初衷就是为了在发生异常的情况下不影响程序的运行，如果main方法继续上抛异常给JVM，则势必会导致程序的退出
  • 一个方法中如果将异常上报了，则该方法运行结束，不会再运行后续的代码

throw关键字

  • 在方法体中抛出主动抛出异常
  • 语法格式

使用try...catch语句和进行异常的捕捉

  • 自己处理这个异常

语法格式

  • catch捕捉到的异常必须是发生的或接受到的异常，或者是以上异常的父类
  • catch后建议只精确的捕捉一个异常，针对某一个异常进行异常处理
  • catch后跟多个异常时可以用｜来分隔开（JDK8）
  • 一个try后可以跟多个catch，处理多个异常
    • 多个catch时会从上到下执行，那么必须遵守多个异常从小到大的原则，一个异常不能被多次捕获

异常对象的常用方法

public String getMessage()

Returns the detail message string of this Throwable instance (which may be null).

  • 获取异常简单的描述信息
    • 这个信息其实就是构造方法中的字符串参数

printStackTrace()

Prints this throwable and its backtrace to the standard error stream. This method prints a stack trace for this Throwable object on the error output stream that is the value of the field System

  • 获取异常追踪的堆栈信息
  • Java后台打印异常堆栈追踪信息时，采用的了异步线程的方式打印
    • 专门有一个线程用于打印这个信息
  • 建议在所有异常处理时都打印一下异常追踪的堆栈信息，辅助我们处理异常
