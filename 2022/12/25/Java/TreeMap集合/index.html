Java学习-TreeMap集合

TreeMap集合

TreeMap的存储

  • TreeMap是实现了Map接口下的SortedMap接口
  • TreeMap集合中元素的存储特点
    • 无序不可重复
    • 集合key部分的元素会自动按照从大到小的顺序排序
  • TreeMap集合底层是一个二叉树
  • 放到TreeSet集合中的元素等同于放到TreeMap集合的key部分
    • 那么直接按照集合中的元素进行排序

自平衡二叉树

  • 遵循左孩子<根节点<右孩子
  • TreeSet和Treemap集合采用的是中序遍历
    • 左孩子->根节点->右孩子
  • 因此我们得到，实现TreeMap或者TreeSet集合中比较功能有两种方法
    1. 该类实现Comparable接口
      • 一个类只能实现一次Comparable接口，其比较规则不能再改变
    2. 重写编写一个Comparator类实现比较器
      • 可以编写多个规则的多个比较器的类，使用不同的比较规则就可以传不同的比较器到集合中

Comparable接口

Comparable接口的实现

  • 放在TreeMap或者TreeSet集合中的元素需要实现java.lang.Comparable接口

    • 这两个集合通过Comparable接口中compareTo()方法进行排序

  • 设计类如果没有实现Comparable接口

    • 会抛出ClassCastException异常

  • 直接让设计的类实现（implements）Comparable接口即可

    • 然后在类中重写compareTo()方法
    • 在实现接口时可以使用泛型，避免之后在重写方法时还需要向下转型
  • 实现Comparable接口

Comparable比较规则

  • compareTo()方法

    • obj1.compareTo(Object obj2)
    • 当obj1的某个属性值大于obj2的属性值时，返回正数
    • 当obj1的某个属性值等于obj2的属性值时，返回0
    • 当obj1的某个属性值小于obj2的属性值时，返回负数

  • this-obj就是升序，obj-this就是降序

  • 更复杂的

    • 按照年龄降序排列，年龄相同的按照姓名升序排列

Comparator比较器

  • 实现TreeSet和TreeMap中比较的第二种方法：单独实现比较器
  • 比较器实现的是java.util.Comparator，比较接口Comparable是java.lang包下的
  • 当比较规则比较多变时，可以编写多个比较器
    • 需要哪种比较规则时，就将某一种比较器作为参数传入到集合中

TreeSet的构造方法

public TreeSet()

  • 有无参的构造方法

public TreeSet(Comparator<? super E> comparator)

  • 有有参的构造方法
    • 参数需要一个比较器
  • 比较器Comparator是一个单独的类
    • 而不是类似实现Comparable接口时，只需要原本的类实现该接口即可

Comparator比较器的实现

  • 单独编写一个比较器的类来实现Comparator接口
    • 这个比较器是专门针对需要比较的类（即存储于TreeMap或者TreeSet中的元素)来实现的
    • 并不能在多个类之间通用

Comparator比较器的匿名类实现

  • 直接通过匿名类的方式来创造一个比较器
    • 一个比较器只用于一个类的比较，没有必要做到复用
  • 匿名类不需要implements关键字
    • 在匿名类中直接重写compare()方法

Collections工具类常用方法

public static <T extends Comparable<? super T>> void sort(List<T> list)

Sorts the specified list into ascending order, according to the natural ordering of its elements. All elements in the list must implement the Comparable interface. Furthermore, all elements in the list must be mutually comparable (that is, e1.compareTo(e2)must not throw a ClassCastException for any elements e1 and e2 in the list).

  • 用于对集合中的元素进行排序
    • 待排序的元素需要实现 Comparable接口
  • public static <T> void sort(List<T> list, Comparator<? super T> c)
  • 类似的，也可以通过参数的方式传入一个比较器用于比较
