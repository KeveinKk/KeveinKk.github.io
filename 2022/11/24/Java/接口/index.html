接口

接口的概念

  • 接口也是一个引用数据类型

    • 虽然接口不是一个类，但它编译之后仍然生成一个.class的字节码文件

  • 接口是完全抽象的

    • 接口是一种特殊的抽象类
    • 抽象类是半抽象的

  • 接口的语法格式

    • 类的定义

    • 抽象类的定义

  • 接口支持多继承，一个接口可以继承多个接口

  • 接口中只包含两个部分：常量 以及 抽象方法

  • 接口中所有的元素都是由public修饰的

    • 在接口中定义方法时，public和abstract两个关键字可以修饰不写
    • 在接口中声明变量时，publi static final三个关键字不写，并且必须手动赋初始值
      • 通过接口直接访问

接口的实现

  • 类和接之间的继承关系称作实现，关键字implements

    • 类和类之间叫做继承，关键字extends

    • 抽象类实现接口时，可以不用实现接口中的方法

      • 抽象类中运行存在抽象方法

    • 非抽象类实现接口时，必须要实现接口中每一个 的方法

    • 在实现接口中的方法时，类中方法的访问控制权限必须是public

      • 子类继承父类的方法，访问权限只能更高，不能更低

接口的多态

  • 接口的多态

  • ```java
    A a = new B();//父类型的引用指向子类型的对象
    a.sum(10,20);

  • 接口的类型转换
    • 向下类型转换
      • 同样需要使用instanceof来判断
    • 强制类型转换
      • 可以两个没有继承关系的接口进行相互转化
      • 编译不会报错，但是运行过程中可能会抛出ClassCastException异常
      • 在类的类型转换中，无论是向上转型还是向下转型，必须要求两个类有继承关系

继承和实现同时存在

  • extends关键字在前，implements关键字在后

  • 接口通常提取的是行为动作

  • 一个类实现一个接口，就代表能够完成该接口的动作

  • 举一个例子

接口的作用

  • 面向抽象编程等价于面向接口编程
  • 有了接口就可以插拔，可插拔代表拓展力强、低耦合度
  • 接口的使用离不开多态
  • A has a B则B是A的一个属性
  • A is a B则A是B的子类（继承）
  • 举一个例子：顾客去餐厅可以点餐，既可以点中餐也可以点西餐

接口和抽象类的区别

  1. 抽象
    • 抽象类是半抽象的：可以在里面定义非抽象的方法，也可以定义普通的成员变量
    • 接口是纯抽象的：在接口中定义的方法都是抽象方法，在接口中声明的变量都是静态常量
  2. 构造方法
    • 抽象类拥有构造方法
    • 接口中没有构造方法
  3. 继承
    • 抽象类只能单继承：一个类只能继承一个抽象类
    • 接口支持多继承：一个类可以实现多个接口
  4. 使用情况
    • 抽象类是对行为和数据的抽象，使用的较少
    • 接口是对行为的抽象
